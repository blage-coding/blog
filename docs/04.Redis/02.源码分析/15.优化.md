# 优化

## 键值设计

[业务名称\]：[数据名称]：[id]

预防出现BigKey

Hash的entry数量不要超过1000，将大的hash打散成小的哈希。

## 持久化

一般关闭RDB采用AOF，因为RDB安全性得不到保证，近几分钟的数据可能没有备份上，优点在于体积小适合数据备份。AOF适合做数据恢复。
